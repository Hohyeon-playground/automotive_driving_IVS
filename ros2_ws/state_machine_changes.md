# State Machine Node 수정 내역

## 개요
ArUco 마커와 손 제스처 기반의 **5단계 상태 머신**으로 로봇의 자동 접근·교환·복귀를 제어한다.
실제 구동 테스트 과정에서 발견된 **버그 수정**과 **동작 안정화**를 중심으로 코드를 개선했다.

### 상태 흐름
```
① 원거리 대기·접근 → ② 근거리 대기 → ③ 접근 → ④ 교환 대기 → ⑤ 복귀 → (다시 ①)
   (발 마커 전진)     (1번 제스처)   (발 마커) (2번 제스처)  (홈 마커)
```

---

## 주요 수정 사항

### 1. 마커 미감지 시 "정지" → "탐색 주행" 으로 변경
- **문제**: 발 마커/홈 마커가 화면에서 사라지면 로봇이 그 자리에 **멈춰버려** 마커를 다시 찾지 못함
- **수정**: 마커가 안 보일 때 `SEARCH_STEER(-1.0)` 로 **선회 주행**하며 마커를 탐색
  - ① 원거리: `MODE_STOP` → `MODE_FORWARD + 탐색 조향`
  - ⑤ 복귀: `MODE_ROTATE` → `MODE_FORWARD + 탐색 조향`

### 2. 제스처 카운터 오작동 수정 (가장 중요)
- **문제**: 제스처 노드가 손이 없을 때 `0` 을 보내는데, 기존 코드는 `0` 수신 시 **카운터를 리셋**
  → 사용자가 제스처를 유지해도 중간에 `0` 이 섞이면 확정이 안 됨
- **수정**: `data=0` 은 **무시**(카운터 유지), `1`/`2` 같은 유효 제스처만 누적
  → 제스처 인식 안정성 대폭 향상

### 3. 조향 방향 부호 수정
- **문제**: `calc_steer()` 에서 부호가 반대(`-tvec_x`)라 마커 **반대쪽으로 조향**
- **수정**: `+tvec_x` 로 변경해 마커 방향으로 정상 조향

### 4. 안전 초기값 / None 체크 추가 (방어 코드)
- `tvec_z` 초기값 `0.0` → **`999.0`**: 포즈 수신 전에 "거리 0" 으로 오인해 즉시 정지하던 문제 방지
- `marker_fresh()` 에 **`marker_id is not None`** 체크 추가: 첫 프레임 None 비교 오류 방지

### 5. 문법 오류 수정
- `on_pose` 함수 정의에 **콜론(`:`) 누락** → 추가 (실행 불가 버그)

### 6. 임계값(파라미터) 튜닝
| 파라미터 | 기존 | 변경 | 의미 |
|---|---|---|---|
| `STOP_THRESHOLD` | 0.15 m | **0.20 m** | ③→④ 교환 정지 거리 (실측 조정) |

## 적용 효과 요약
- ✅ 마커를 놓쳐도 **스스로 탐색**해 복귀 가능
- ✅ 제스처 인식이 **끊김 없이 안정적**으로 확정
- ✅ 마커 방향으로 **올바르게 조향**
- ✅ 초기 구동 시 **오동작·즉시 정지** 문제 제거
- ✅ 실행 불가(문법 오류) 버그 해결
