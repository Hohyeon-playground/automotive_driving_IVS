# 작업 정리 — 2026-06-19

> 환경: Raspberry Pi 5 + Ubuntu 24.04 + ROS 2 Jazzy
> 요약: 통합 구동 시 "자꾸 꺼지고/마커 못 잡던" 문제를 전원·카메라 두 축으로 해결. 실행 환경도 정리.

---

## 1. 전원 문제 — "합치면 자꾸 꺼짐"

- **증상**: 노드 합쳐 돌리면 연결 끊기고 Pi가 리부트 (오늘만 8회+, 23초 만에 죽는 부팅도 있었음).
- **진단**:
  - `vcgencmd get_throttled` sticky bit + `journalctl --list-boots` / `last -x`(세션이 전부 `crash`로 끝남)로 **전압 붕괴(브라운아웃)** 확정.
  - `throttled=0x0`인데도 리셋 → 살짝 강하가 아니라 **완전 붕괴(하드리셋)**, 플래그가 매번 초기화됨.
- **실제 원인**: **Pi에 물린 배터리 방전**. (충전된 전원으로 스왑하니 부하 줘도 10분+ 멀쩡, throttled 0x0)
- **교훈**: 테스트 전 **배터리 잔량부터 확인**. 모터 드라이버 5V는 별도 배터리(GND 공통)로 분리.
- **만든 진단 도구**:
  - `brownout_detect.py` — `get_throttled` sticky bit 고속(50Hz) 폴링, ms급 전압강하 포착.
  - `hw_monitor.py` — 입력5V/온도/CPU 실시간 그래프 (추세용).

## 2. 카메라 문제 — "마커를 못 잡음"

- **증상**: aruco가 엉뚱한 카메라를 봐서 `/aruco/pose` 발행 안 됨.
- **원인**:
  1. OpenCV가 **GStreamer 백엔드**로 열려 `cv2.VideoCapture(N)` 인덱스가 `/dev/videoN`과 불일치.
  2. 동일 모델 카메라 2대(시리얼 없음) → 재부팅 시 `/dev/videoN` 번호 reshuffle.
- **해결 (코드 반영)**:
  1. **V4L2 백엔드 강제**: `cv2.VideoCapture(src, cv2.CAP_V4L2)`.
  2. **by-path 고정경로 기본값** (USB 포트 기준이라 재부팅 불변):
     | 노드 | 카메라 | by-path |
     |---|---|---|
     | aruco (마커/아래) | /dev/video0 | `platform-xhci-hcd.1-usb-0:2:1.0-video-index0` |
     | hand_gesture (위/손) | /dev/video2 | `platform-xhci-hcd.0-usb-0:2:1.0-video-index0` |
  3. 두 노드에 `camera_path` 파라미터 추가(우선), 비면 `camera_index` 폴백. → 평소 인자 불필요.
- **확인 도구**: `cam_probe.py <index>` — 콘솔로 마커 검출 실시간 확인(창 불필요).
- **창 주의**: cv2 창은 **Pi 본체 모니터**에서만 뜸. Ubuntu 24.04 Wayland라 SSH(`ssh -X` 포함)는 X 권한으로 안 뜸.

## 3. 실행 환경 정리

- **GPIO**: `ivs`가 `dialout` 그룹 + `/dev/gpiochip4`가 udev로 dialout에 열림 → `sudo chmod` 불필요(스크립트에서 제거, sudo 프롬프트로 멈추던 것도 해소).
- **런처**:
  - `start_robot.sh` — tmux 4칸(aruco/gesture/state/monitor) + motor 윈도우. 카메라 창 ON, `DISPLAY=:0` 자동. **칸 배치 버그 수정**(tiled가 인덱스 순서로 재배치 → 4칸 만든 뒤 인덱스 순서대로 명령 주입). **Pi 데스크톱**에서 실행.
  - `run_robot.ps1` — Windows PowerShell에서 SSH로 노드 4창(headless). 영상 창 안 뜸.
  - VSCode Remote-SSH 쓰면 그냥 **통합 터미널 여러 개**(이미 Pi bash)로 노드별 실행이 제일 간단.

## 4. 모터 회전 감속 (기존 코드, 오늘 변경 아님)

`motor_node.py`의 `drive_speed()`에 이미 구현됨:
```python
return max(DRIVE_MIN, DRIVE_BASE - DRIVE_TURN_SLOW * abs(steer_angle))
```
- `DRIVE_BASE=0.50`(직진 순항), `DRIVE_TURN_SLOW=0.15`(조향 비례 감산), `DRIVE_MIN=0.35`(스톨 방지 바닥값).
- 속도 = `max(0.35, 0.50 - 0.15×|조향각|)` → **꺾을수록 감속**(회전 반경 축소/언더스티어 억제).
- 출발 킥: 조향 출발 `KICK_STEER=0.85`, 직진 출발 `KICK_STRAIGHT=0.70` (정지 마찰 극복).
- 튜닝하려면 이 상수들만 조정.
