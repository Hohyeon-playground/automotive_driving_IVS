# Follow-Tool 프로젝트 명세 (최종)

> 작업자가 신호를 보내면, 공구함을 실은 로봇이 작업자에게 와서 공구를 주고받고, 원래 자리로 돌아간다.

---

## 1. 역할 정의

### 손 모션 (위 카메라 + Hailo-8)
- 트리거 2가지만 수행. 조향 / 속도 제어에 관여 없음
- **1번 제스처** → 근거리 진입 후 본격 접근 트리거 (② → ③ 전환)
- **2번 제스처** → 복귀 시작 트리거 (완료 신호, 미확정)

### ArUco 마커 (아래 카메라 + OpenCV)
- AI 연산 없음, CPU만 사용
- `solvePnP`로 `tvec` 추출
  - `tvec_x` : 마커 좌우 편차 → 서보 조향각 계산에 사용
  - `tvec_z` : 마커까지 정면 거리 → 정지 판단에 사용
- 마커 ID로 역할 구분
  - **발 마커 ID** : 접근 단계 (② 접근)
  - **홈 마커 ID** : 복귀 단계 (④ 복귀)

### 서보 모터
- 조향 전담
- `tvec_x` 편차에 따라 조향각 설정
- PID 없음 — zone 기반 또는 단순 비례 제어

### DC 모터
- 속도 전담
- 전진 / 정지 / 제자리 회전 3가지 모드
- ON/OFF 또는 PWM 제어

## 2. 상태 흐름

```
① 원거리 대기·접근
   아래 카메라: 발 마커 원거리 감지 감시
   DC 정지 / 서보 중립
   발 마커 감지 시 → DC 전진 + tvec_x → 서보 조향으로 자동 접근
   발 마커 tvec_z < 근거리 임계값(손 인식 가능 거리) → 정지
   │
   │  근거리 임계값 진입
   ▼
② 근거리 대기
   위 카메라 감시 → 1번 제스처 N프레임 연속 감지
   DC 정지 / 서보 중립
   │
   │  1번 제스처 감지
   ▼
③ 접근
   아래 카메라: 발 마커 tvec_x → 서보 조향
   DC 전진
   발 마커 tvec_z < 정지 임계값 → 정지
   │
   │  발 마커 tvec_z < 정지 임계값
   ▼
④ 교환 대기
   위 카메라 감시 → 2번 제스처 감지
   DC 정지 / 서보 중립
   │
   │  2번 제스처 감지
   ▼
⑤ 복귀
   DC 제자리 회전 → 홈 마커 탐색
   홈 마커 감지 → DC 전진 + tvec_x → 서보 조향
   홈 마커 tvec_z < 임계값 → 정지
   │
   │  홈 마커 tvec_z < 임계값
   ▼
① 원거리 대기 (루프)
```

## 3. ROS 노드 구성

| 노드 | 역할 | 입력 | 출력 토픽 |
|------|------|------|-----------|
| `hand_gesture_node` | 위 카메라 + Hailo-8로 제스처 감지 | 위 카메라 영상 | `/gesture` |
| `aruco_node` | 아래 카메라로 마커 pose 계산 | 아래 카메라 영상 | `/aruco_pose` |
| `state_machine_node` | 상태 판단 + 전환 + 명령 생성 | `/gesture`, `/aruco_pose` | `/motor_cmd` |
| `motor_node` | 서보/DC 모터 실제 구동 | `/motor_cmd` | 하드웨어 |

### 토픽 명세

```
/gesture    std_msgs/Int32                    # data: 0=없음  1=호출  2=완료
                                              # std_msgs 사용 이유: 손모션팀 Pi와 인터페이스 공유 불필요
/aruco_pose motor_interfaces/ArucoPose
              int32   id                      # 발 마커 ID / 홈 마커 ID
              float32 tvec_x                  # 좌우 편차 (m)
              float32 tvec_z                  # 정면 거리 (m)
/motor_cmd  motor_interfaces/MotorCmd
              int32   mode                    # 0=stop  1=forward  2=rotate
              float32 steer_angle             # 서보 각도 (도)
```

## 4. 시각화 1 — 상태 흐름 × 노드 역할

각 상태에서 어떤 노드가 무엇을 수행하는지.  
`[ON]` = 활성 &nbsp; `[--]` = 비활성(대기)

```
─────────────────────────────────────────────────────────────────────────────────
  ① 원거리 대기·접근
─────────────────────────────────────────────────────────────────────────────────
  hand_gesture_node   [--]  대기
  aruco_node          [ON]  발 마커 원거리 감시 | 감지 시 solvePnP → tvec 계산
                            발행: /aruco_pose { id, tvec_x, tvec_z }
  state_machine_node  [ON]  마커 미감지: DC 정지
                            마커 감지  : tvec_x → steer_angle, DC 전진
                            tvec_z < 근거리 임계값 → 정지 + ② 전환
  motor_node          [ON]  마커 감지 전: DC 정지 / 서보 중립
                            마커 감지 후: DC 전진 / 서보 steer_angle 적용

  전환 조건 ── 발 마커 tvec_z < 근거리 임계값(손 인식 가능 거리) ──────────── ▼ ②

─────────────────────────────────────────────────────────────────────────────────
  ② 근거리 대기
─────────────────────────────────────────────────────────────────────────────────
  hand_gesture_node   [ON]  위 카메라 감시 | 1번 제스처 N프레임 감지
                            발행: /gesture data=1
  aruco_node          [--]  대기
  state_machine_node  [ON]  /gesture 구독 | data == 1 → ③ 전환
  motor_node          [ON]  DC 정지 / 서보 중립

  전환 조건 ── 1번 제스처 N프레임 연속 감지 ──────────────────────────────── ▼ ③

─────────────────────────────────────────────────────────────────────────────────
  ③ 접근
─────────────────────────────────────────────────────────────────────────────────
  hand_gesture_node   [--]  대기
  aruco_node          [ON]  발 마커 인식 | solvePnP → tvec 계산
                            발행: /aruco_pose { id, tvec_x, tvec_z }
  state_machine_node  [ON]  /aruco_pose 구독 | tvec_x → steer_angle 계산
                            tvec_z < 정지 임계값 → 정지 + ④ 전환
  motor_node          [ON]  DC 전진 / 서보 steer_angle 적용

  전환 조건 ── 발 마커 tvec_z < 정지 임계값 ──────────────────────────────── ▼ ④

─────────────────────────────────────────────────────────────────────────────────
  ④ 교환 대기
─────────────────────────────────────────────────────────────────────────────────
  hand_gesture_node   [ON]  위 카메라 감시 | 2번 제스처 감지
                            발행: /gesture data=2
  aruco_node          [--]  대기
  state_machine_node  [ON]  /gesture 구독 | data == 2 → ⑤ 전환
  motor_node          [ON]  DC 정지 / 서보 중립

  전환 조건 ── 2번 제스처 감지 ───────────────────────────────────────────── ▼ ⑤

─────────────────────────────────────────────────────────────────────────────────
  ⑤ 복귀
─────────────────────────────────────────────────────────────────────────────────
  hand_gesture_node   [--]  대기
  aruco_node          [ON]  홈 마커 탐색 | solvePnP → tvec 계산
                            발행: /aruco_pose { id, tvec_x, tvec_z }
  state_machine_node  [ON]  /aruco_pose 구독
                            마커 미감지 → rotate 명령
                            마커 감지   → tvec_x → steer_angle 계산
                            tvec_z < 임계값 → 정지 + ① 전환
  motor_node          [ON]  탐색 중: DC 제자리 회전
                            마커 감지: DC 전진 / 서보 steer_angle 적용

  전환 조건 ── 홈 마커 tvec_z < 임계값 ────────────────── ▼ ① 원거리 대기 (루프)
─────────────────────────────────────────────────────────────────────────────────
```

## 5. 시각화 2 — 노드 간 관계

ROS 토픽을 통한 노드 간 데이터 흐름.

```
  [위 카메라 + Hailo-8]                        [아래 카메라 (OpenCV)]
   손 모션 AI 추론                               ArUco 마커 인식 (CPU only)
          │                                              │
          ▼                                              ▼
  ┌──────────────────────┐                ┌──────────────────────────┐
  │  hand_gesture_node   │                │       aruco_node         │
  │                      │                │                          │
  │  Hailo-8 추론 수신   │                │  발 마커 / 홈 마커 인식  │
  │  1번 제스처: 호출    │                │  solvePnP → tvec 계산    │
  │  2번 제스처: 완료    │                │  tvec_x (편차)           │
  └──────────┬───────────┘                └────────────┬─────────────┘
             │                                         │
             │  /gesture                               │  /aruco_pose
             │  data: 0 | 1 | 2                        │  { id, tvec_x, tvec_z }
             │                                         │  발 마커 ID / 홈 마커 ID
             └──────────────────┬──────────────────────┘
                                │
                                ▼
                ┌───────────────────────────────┐
                │      state_machine_node        │
                │                               │
                │  ① 대기 → ② 접근             │
                │  → ③ 교환 대기 → ④ 복귀      │
                │                               │
                │  전환 조건 판단               │
                │  /motor_cmd 생성 및 발행       │
                └───────────────┬───────────────┘
                                │
                                │  /motor_cmd
                                │  { mode: stop|forward|rotate,
                                │    steer_angle: float }
                                │
                                ▼
                ┌───────────────────────────────┐
                │          motor_node            │
                └──────────┬────────────┬────────┘
                           │            │
                           ▼            ▼
                    [서보 모터]      [DC 모터]
                    조향 제어        전진 / 정지 / 제자리 회전
                    steer_angle      ON/OFF 또는 PWM
```

## 6. 하드웨어 구성

| 부품 | 역할 |
|------|------|
| Raspberry Pi 5 | 메인 컴퓨터 |
| Hailo-8 (AI HAT+) | 손 모션 AI 추론 전담 |
| USB 카메라 (위) | 위 대각선 방향, 손 인식 전담 |
| USB 카메라 (아래) | 정면 방향, ArUco 마커 인식 전담 |
| 서보 모터 | 조향 |
| DC 모터 | 전진 / 정지 / 회전 |

### 카메라 스탠드 구조

```
  카메라 스탠드 (세로)
  ├── 위 카메라  ↗  (위 대각선, 손 인식)
  └── 아래 카메라 →  (정면, 마커 인식)
  자동차 몸체
```
