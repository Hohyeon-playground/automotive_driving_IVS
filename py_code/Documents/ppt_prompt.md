| USB 카메라 (아래) | ArUco 마커 인식 |
| 서보 모터 | 조향 제어 |
| DC 모터 | 전진 / 정지 / 제자리 회전 |

### 슬라이드 4 — 시나리오 (상태 흐름)

**제목**: 동작 시나리오

4개 상태를 좌→우 또는 위→아래 흐름으로 배치.
각 상태는 색으로 구분:
- ① 대기: 파란색
- ② 접근: 초록색
- ③ 교환 대기: 주황색
- ④ 복귀: 보라색

각 상태 박스 안에 아래 정보 표시:
- **① 대기**: 위 카메라 감시 / 1번 제스처 N프레임 감지 대기 / DC 정지·서보 중립
- **② 접근**: 발 마커 추적(aruco) / 서보 조향 + DC 전진 / tvec_z < 임계값 → 정지
- **③ 교환 대기**: 위 카메라 감시 / 2번 제스처 감지 대기 / DC 정지·서보 중립
- **④ 복귀**: 제자리 회전 → 홈 마커 탐색 / 마커 감지 후 DC 전진 + 서보 조향 / tvec_z < 임계값 → 정지

전환 조건은 화살표 위 또는 아래에 작게 표시:
- ①→②: 1번 제스처 감지
- ②→③: 발 마커 tvec_z < 임계값
- ③→④: 2번 제스처 감지
- ④→①: 홈 마커 tvec_z < 임계값 (루프)

**[이미지 삽입]**: viz1_state_nodes.html 캡처 이미지를 이 슬라이드 하단 또는 다음 슬라이드에 삽입

### 슬라이드 5 — 노드 구성

**제목**: ROS2 노드 구성

4개 노드와 토픽 흐름을 다이어그램으로 표시:

```
[위 카메라 + Hailo-8]          [아래 카메라 (OpenCV)]
        ↓                               ↓
hand_gesture_node              aruco_node
/gesture 발행                  /aruco_pose 발행
{type: 0|1|2}                  {id, tvec_x, tvec_z}
        ↘                             ↙
            state_machine_node
            (상태 판단 + 전환)
                    ↓
              /motor_cmd
         {mode, steer_angle}
                    ↓
              motor_node
          ↙              ↘
    서보 모터           DC 모터
    (조향)         (전진/정지/회전)
```

각 노드 색상:
- hand_gesture_node: 파란색
- aruco_node: 초록색
- state_machine_node: 보라색
- motor_node: 주황색

**[이미지 삽입]**: viz2_node_relations.html 캡처 이미지를 이 슬라이드 하단 또는 다음 슬라이드에 삽입

## 디자인 가이드

- 전체 컬러 테마: 다크 배경 (#0f172a) + 흰색/밝은 텍스트 권장  
  (또는 흰색 배경의 밝고 깔끔한 테마도 무방)
- 노드 색상은 슬라이드 4·5와 일관성 유지:
  - hand_gesture_node → 파란색 계열
  - aruco_node → 초록색 계열
  - state_machine_node → 보라색 계열
  - motor_node → 주황색 계열
- 폰트: 명확하고 가독성 좋은 산세리프 (Pretendard, Noto Sans KR 등)
- 슬라이드 비율: 16:9
- 코드/토픽명은 고정폭 폰트(monospace)로 표시
