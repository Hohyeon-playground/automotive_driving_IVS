# Follow-Tool 프로젝트 명세서

## 1. 프로젝트 개요

### 한 줄 요약
> 작업자가 신호를 보내면, 공구함을 실은 로봇이 작업자에게 와서 공구를 주고받고, 원래 자리로 돌아간다.

### 문제 정의
작업 현장에서 필요한 공구를 가지러 이동하는 반복 행위는 작업 흐름을 끊고 효율을 낮춘다.
작업자가 자리를 지키면서 공구를 주고받을 수 있도록, 자율 이동 공구함 로봇을 구현한다.

### 핵심 동작 요약
```
보고 → 온다 → 기다린다 → 돌아간다
```

---

## 2. 시스템 구성

### 하드웨어
- Raspberry Pi 5
- Hailo-8 AI 가속기 (AI HAT+)
- RC카 키트 베이스 (DC모터 + 서보모터)
- USB 카메라 2대
  - **위 카메라** (위 대각선 방향): 손 모션 인식 전담, Hailo-8 연결
  - **아래 카메라** (정면 방향): ArUco 마커 인식 전담, OpenCV(CPU)만 사용
- 공구함 트레이

### 동작 환경 가정
- 실내 환경 (교실 수준)
- 호출 거리: 5m 이상 (ArUco 마커 원거리 감지 기반)
- 손 제스처 인식 유효 거리: 실험 측정 필요 (① → ② 전환 기준)
- 홈 마커는 아래 카메라 시야 내 고정 위치에 부착
- 복귀 시 홈 마커 미감지 시 제자리 회전으로 탐색

## 3. 상태 정의

로봇은 아래 5개 상태를 순환한다.

| 상태 | 동작 | 전환 조건 |
|------|------|-----------|
| ① 원거리 대기·접근 | 발 마커 원거리 감지 → 자동 접근 | 발 마커 tvec_z < 근거리 임계값 |
| ② 근거리 대기 | 정지, 위 카메라로 제스처 감시 | 1번 제스처 N프레임 연속 감지 |
| ③ 접근 | 발 마커 추적하며 이동 | 발 마커 tvec_z < 정지 임계값 |
| ④ 교환 대기 | 정지, 위 카메라로 제스처 감시 | 2번 제스처 감지 |
| ⑤ 복귀 | 홈 마커 탐색 후 이동 | 홈 마커 tvec_z < 임계값 |

### 상태 전환 흐름
```
① 원거리 대기·접근
   │ 발 마커 감지 → 자동 접근
   │ tvec_z < 근거리 임계값(손 인식 가능 거리)
   ▼
② 근거리 대기  (정지 후 위 카메라로 1번 제스처 감시)
   │ 1번 제스처 N프레임 연속
   ▼
③ 접근  (발 마커 tvec_x → 서보 조향 / tvec_z → 정지 판단)
   │ 발 마커 tvec_z < 정지 임계값
   ▼
④ 교환 대기  (위 카메라로 2번 제스처 감시)
   │ 2번 제스처 감지
   ▼
⑤ 복귀  (제자리 회전 → 홈 마커 탐색 → ArUco 서보잉)
   │ 홈 마커 tvec_z < 임계값
   ▼
① 원거리 대기  (루프)
```

## 4. 구현 목록

### 핵심 기능 (Must Have)

#### 4-1. 원거리 마커 감지·자동 접근 (상태 ①)
- 아래 카메라 + OpenCV aruco 모듈
- 발 마커 원거리 감지 → tvec_x 기반 서보 조향 + DC 전진으로 자동 접근
- `tvec_z < 근거리 임계값` (손 인식 가능 거리) → 정지 → ② 전환
- 마커 미감지 시 DC 정지 유지

#### 4-2. 손 제스처 인식 (트리거 전용)
- 위 카메라 + Hailo-8 hef 모델
- 제스처 2가지만 인식, 조향/속도 제어에 관여 없음
  - **1번 제스처**: 접근 트리거 (② → ③ 전환)
  - **2번 제스처**: 완료 트리거 (④ → ⑤ 전환, 미확정)
- 오인식 방지: 1번 제스처는 연속 N프레임 이상 유지 시에만 전환

#### 4-3. ArUco 마커 추적 (접근 + 복귀 공통)
- 아래 카메라 + OpenCV aruco 모듈 (AI 연산 없음)
- `solvePnP`로 tvec 추출
  - `tvec_x`: 좌우 편차 → 서보 조향각 계산
  - `tvec_z`: 정면 거리 → 정지 판단
- 마커 ID로 역할 구분 (발 마커 ID ≠ 홈 마커 ID)

#### 4-4. 접근 제어 (상태 ③)
- 발 마커 tvec_x → 서보 조향 (zone 기반 또는 단순 비례, PID 없음)
- DC 모터 전진 (ON/OFF 또는 PWM)
- 발 마커 tvec_z < 정지 임계값 → 정지 → ④ 전환

#### 4-5. 교환 감지 (상태 ④)
- 위 카메라 + Hailo-8
- 2번 제스처 감지 → ⑤ 복귀 트리거
- 로드셀/센서 미사용

#### 4-6. 홈 복귀 (상태 ⑤)
- DC 모터 제자리 회전으로 홈 마커 탐색
- 홈 마커 감지 후: tvec_x → 서보 조향, DC 전진
- 홈 마커 tvec_z < 임계값 → 정지 → ① 원거리 대기

---

### 추가 기능 (Nice to Have)

#### 4-6. 장애물 회피
- 센서: 초음파 센서 (전방)
- 동작: 접근 중 전방 N미터 이내 감지 시 감속/정지
- 우선순위: 핵심 루프 완성 후 구현

## 5. 예외 처리

| 상황 | 처리 방법 |
|------|-----------|
| ① 원거리 접근 중 발 마커 소실 | 마커 재감지 시까지 DC 정지 유지 |
| ② 근거리 대기 중 1번 제스처 오인식 | N프레임 연속 조건으로 필터링 |
| ③ 접근 중 발 마커 잠깐 소실 | 디바운싱으로 ③ 상태 유지 |
| ⑤ 복귀 중 홈 마커 미발견 | 제자리 회전 반복 |
