# Follow-Tool 구현 덩어리 명세서

## 1. 손 제스처 인식 (트리거 전용)

### 목적
1번 제스처: 대기 → 접근 전환 트리거
2번 제스처: 교환 대기 → 복귀 전환 트리거
조향/속도 제어에는 관여하지 않는다.

### 구현 방식
- **위 카메라 + Hailo-8 hef 모델** (기성 모델 사용)
- 제스처 분류 Hailo-8에서 처리
- ROS 노드: `hand_gesture_node`
  - 퍼블리시: `/gesture { type: int }` (0=없음, 1=호출, 2=완료)

### 트리거 조건
- **1번 제스처**: 연속 N프레임 이상 유지 시에만 상태 전환 (오인식 방지)
- **2번 제스처**: 감지 즉시 트리거 (N프레임 여부 추후 결정)

---

## 2. ArUco 마커 추적 (접근 + 복귀 공통 모듈)

### 목적
이동 중 마커를 기준으로 서보 조향각과 정지 시점을 결정한다.
② 접근(발 마커)과 ④ 복귀(홈 마커) 모두 동일 코드, 마커 ID만 다름.

### 구현 방식
- **아래 카메라 + OpenCV aruco 모듈** (AI 연산 없음, CPU only)
- `estimatePoseSingleMarkers` 또는 `solvePnP`로 `rvec`, `tvec` 추출
  - `tvec_x`: 마커 좌우 편차 → 서보 조향값 계산에 사용
  - `tvec_z`: 마커까지 정면 거리 → 정지 판단에 사용
- ROS 노드: `aruco_node`
  - 퍼블리시: `/aruco_pose { id: int, tvec_x: float, tvec_z: float }`

### 조향 제어
- `tvec_x` 편차에 따라 서보 조향각 설정
- PID 없음 — zone 기반 또는 단순 비례 제어
- D 항 미사용 (카메라 프레임 노이즈 증폭 우려)

### 정지 판단
- `tvec_z < 임계값` → 정지 및 다음 상태 전환
- 임계값: 실제 주행 테스트 후 상수로 확정

### 마커 구분
- 발 마커 ID ≠ 홈 마커 ID (aruco 마커 ID 필드로 구분)

---

## 3. 접근 제어 (상태 ②)

### 목적
발 마커 방향으로 이동하여 작업자 앞에 정지한다.

### 구현 방식
- 입력: `aruco_node`의 `/aruco_pose` (발 마커 tvec_x, tvec_z)
- 조향: `tvec_x` → 서보 조향각 제어
- 속도: DC 모터 전진 (ON/OFF 또는 PWM)
- ROS 노드: `state_machine_node` 내 ② 상태 처리
  - 퍼블리시: `/motor_cmd { mode: forward, steer_angle: float }`

### 전환 조건 (② → ③)
- 발 마커 `tvec_z < 임계값` → 정지 → ③ 교환 대기

---

## 4. 교환 감지 (상태 ③)

### 목적
작업자가 공구 교환 완료를 제스처로 알리면 복귀를 시작한다.

### 구현 방식
- **위 카메라 + Hailo-8** (1번과 동일 노드 `hand_gesture_node` 재활용)
- `/gesture { type: 2 }` 수신 → ④ 복귀 트리거
- 로드셀/IR 센서 미사용

---

## 5. 홈 복귀 (상태 ④)

### 목적
교환 완료 후 홈 위치로 복귀하여 대기 상태로 전환한다.

### 구현 방식
- **아래 카메라 + OpenCV aruco** (2번 ArUco 마커 추적과 동일 코드, ID만 다름)
- ROS 노드: `state_machine_node` 내 ④ 상태 처리

### 홈 마커 탐색
- 복귀 시작 시 DC 모터 제자리 회전
- 홈 마커 감지되면 즉시 서보 조향 + 전진 시작

### 복귀 완료 조건
- 홈 마커 `tvec_z < 임계값` → 정지 → ① 대기 (루프 완결)

### 예외 처리
- N초 경과 후에도 홈 마커 미발견 → 제자리 회전 반복

---

## 6. 장애물 회피 (추가 기능)

### 목적
접근 중 전방 장애물을 감지하여 충돌을 방지한다.

### 구현 방식
- **초음파 센서** (전방)
- 전방 N미터 이내 감지 시 감속/정지

### 우선순위
- 핵심 루프 완성 후 구현

---

## 전체 요약

| 상태 | 구현 덩어리 | 핵심 기술 | ROS 노드 |
|------|------------|----------|----------|
| ①→② 트리거 | 손 제스처 인식 | 위 카메라 + Hailo-8 hef | `hand_gesture_node` |
| ② 접근 | ArUco 마커 추적 | 아래 카메라 + OpenCV solvePnP | `aruco_node` |
| ③→④ 트리거 | 손 제스처 인식 | 위 카메라 + Hailo-8 hef | `hand_gesture_node` |
| ④ 복귀 | ArUco 마커 추적 | 아래 카메라 + OpenCV solvePnP | `aruco_node` |
| 전체 판단 | 상태 머신 | `/gesture` + `/aruco_pose` 구독 | `state_machine_node` |
| 실행 | 모터 제어 | 서보 + DC | `motor_node` |
