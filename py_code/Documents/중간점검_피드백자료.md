## 1. 프로젝트 최종 목표

### 한 줄 요약
> 작업자가 신호를 보내면, 공구함을 실은 로봇이 작업자에게 와서 공구를 주고받고, 원래 자리로 돌아간다.

### 문제 정의
작업 현장에서 필요한 공구를 가지러 이동하는 반복 행위는 작업 흐름을 끊고 효율을 낮춘다.
작업자가 자리를 지키면서 공구를 주고받을 수 있도록, **자율 이동 공구함 로봇**을 구현한다.

### 핵심 동작 요약
```
보고 → 온다 → 기다린다 → 돌아간다
```

### 시스템 구성

| 구성 요소 | 역할 |
|-----------|------|
| Raspberry Pi 5 | 메인 컴퓨터 |
| Hailo-8 (AI HAT+) | 손 모션 AI 추론 전담 |
| USB 카메라 (위) | 손 제스처 인식 전담 |
| USB 카메라 (아래) | ArUco 마커 인식 전담 |
| 서보 모터 | 조향 제어 |
| DC 모터 | 전진 / 정지 / 제자리 회전 |

### 상태 흐름

```
① 대기  ──[1번 제스처]──▶  ② 접근  ──[발 마커 근접]──▶  ③ 교환 대기  ──[2번 제스처]──▶  ④ 복귀  ──[홈 마커 근접]──▶  ① 대기
```

### 핵심 구현 우선순위

```
1단계 (핵심 루프) ← 현재 여기
  ├─ hand_gesture_node  (1번/2번 제스처 → /gesture 토픽)
  ├─ aruco_node         (발/홈 마커 tvec_x, tvec_z → /aruco_pose 토픽)
  ├─ state_machine_node (4개 상태 전환 판단 → /motor_cmd 토픽)
  └─ motor_node         (서보 조향 + DC ON/OFF/PWM)

2단계 (추가 기능)
  └─ 장애물 회피 (초음파 센서, 핵심 루프 완성 후)
```

## 4. 업무 분장

| 팀 | 팀원 | 담당 |
|----|------|------|
| **Marker팀** | 호현, 태경, 승준 | `aruco_node` 구현 (ArUco 마커 인식, solvePnP, tvec 계산) |
| **손모양팀** | 서진, 준상, 소은 | `hand_gesture_node` 구현 (Hailo 모델 연동, 제스처 분류, /gesture 퍼블리시) |
| **하드웨어팀** | 재훈, 은진, 호현 | 차량 조립·배선, 카메라 거치 완성, `motor_node` 구현 (서보/DC 제어) |

> `state_machine_node` 및 통합 테스트는 전원 참여

## 5. ROS2 노드 구성 (참고)

| 노드 | 담당 | 입력 | 출력 토픽 |
|------|------|------|-----------|
| `hand_gesture_node` | 손모양팀 | 위 카메라 영상 | `/gesture { type: 0\|1\|2 }` |
| `aruco_node` | Marker팀 | 아래 카메라 영상 | `/aruco_pose { id, tvec_x, tvec_z }` |
| `state_machine_node` | 전원 | `/gesture`, `/aruco_pose` | `/motor_cmd { mode, steer_angle }` |
| `motor_node` | 하드웨어팀 | `/motor_cmd` | 서보/DC 하드웨어 |
